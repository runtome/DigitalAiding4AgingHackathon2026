# DigitalAiding4AgingHackathon2026

## ระบบประเมินความคล่องแคล่วของแขนส่วนบน ด้วย AI

### เกี่ยวกับโปรเจกต์

ระบบนี้เป็นเครื่องมือประเมินความคล่องแคล่วของแขนส่วนบน สำหรับผู้สูงอายุหรือผู้ที่มีความเสี่ยงต่อภาวะ Sarcopenia (กล้ามเนื้อเสื่อม) และ Learned Non-Use (LNU — การหลีกเลี่ยงการใช้แขนข้างใดข้างหนึ่งโดยไม่รู้ตัว) พัฒนาสำหรับการแข่งขัน DigitalAiding4Aging Hackathon 2026

ผู้ทดสอบยืนหน้ากล้องเว็บแคม และเอื้อมมือแตะโซนต่าง ๆ ที่ไฮไลต์บนตาราง 3×3 ระบบจะวิเคราะห์:

- **เวลาตอบสนอง (Reaction Time)** — ความเร็วในการเคลื่อนมือไปยังเป้าหมาย
- **ความแม่นยำ (Accuracy)** — สัดส่วนของเป้าหมายที่แตะสำเร็จ
- **คุณภาพการเคลื่อนไหว (Movement Quality)** — ความราบรื่น ประสิทธิภาพเส้นทาง และค่าสั่นสะเทือน
- **ความเสี่ยง LNU** — ความไม่สมมาตรระหว่างแขนซ้ายและขวา
- **อายุมอเตอร์ (Motor Age)** — การประมาณอายุเชิงการเคลื่อนไหวเทียบกับมาตรฐาน

7. **เริ่มใหม่** — คลิก **🔄 เริ่มใหม่** เพื่อทดสอบซ้ำ

### คำอธิบายตัวชี้วัดทางคลินิก

#### 1. LNU Risk (ความเสี่ยง Learned Non-Use)

LNU คือภาวะที่ผู้ป่วยหลีกเลี่ยงการใช้แขนข้างใดข้างหนึ่ง (มักเกิดหลังโรคหลอดเลือดสมอง) โดยไม่รู้ตัว ระบบคำนวณจาก:

- **ความไม่สมมาตรของการใช้งาน (50%)** — สัดส่วนการตีเป้าหมายต่างกันระหว่างสองมือ
- **ความไม่สมมาตรของ RT (30%)** — ความต่างของเวลาตอบสนองเฉลี่ยระหว่างสองมือ
- **ความไม่สมมาตรของคุณภาพ (20%)** — ความต่างของคะแนน Composite ระหว่างสองมือ

ค่า LNU 0–100: **ต่ำ (<33)** / **ปานกลาง (33–67)** / **สูง (>67)**

#### 2. Motor Age (อายุมอเตอร์)

การประมาณ "อายุทางการเคลื่อนไหว" จากประสิทธิภาพ:

```
Motor Age = 20 + (100 − composite_score) / 100 × 65 + tremor_adjustment
```

เปรียบเทียบกับอายุจริงของผู้ทดสอบ ค่า Motor Age ต่ำกว่าอายุจริง = ดี

#### 3. Tremor Power (ค่าสั่นสะเทือน)

ใช้การแปลง FFT (Fast Fourier Transform) บนเส้นทางการเคลื่อนไหวที่ Resample เป็น 10 Hz วัดสัดส่วนพลังงานในย่านความถี่ **3–7 Hz** ซึ่งเป็นช่วงของการสั่น (tremor) ที่พบใน Parkinson และภาวะอื่น ๆ

ค่า 0 = ไม่มีสั่น / ค่า 100 = สั่นมาก

#### 4. Jerk Score (ความราบรื่นของการเคลื่อนไหว)

คำนวณจาก **Normalized Jerk Score (NJS)** ซึ่งเป็น Integral ของขนาด Jerk (อนุพันธ์อันดับสามของตำแหน่ง) ที่ปรับด้วยระยะเวลาและระยะทาง:

- NJS สูง = การเคลื่อนไหวกระตุก / ไม่ราบรื่น
- Jerk Score (0–100) = `100 − log(1 + NJS) × 15` — คะแนนสูง = ราบรื่น

### สถาปัตยกรรมของระบบ

ระบบแบ่งออกเป็น 5 โมดูลหลัก:

```
app.py          — Gradio UI, event handlers, game state wiring
src/tracker.py  — HandTracker (MediaPipe Holistic), grid cell mapping
src/game_engine.py     — GameState machine, hit/timeout detection
src/motion_analyzer.py — Post-game analysis (speed, accuracy, quality, LNU, motor age)
src/visualizer.py      — OpenCV frame overlays, Plotly charts
src/report_generator.py — CSV / PDF export
```

กระแสข้อมูลหลัก: กล้องเว็บแคม → `HandTracker` → `GameEngine` → `draw_overlay` → UI (Gradio streaming) เมื่อจบการทดสอบ → `MotionAnalyzer` → กราฟ Plotly → รายงาน PDF/CSV
